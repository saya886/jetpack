# Jetpack 项目任务 Skill

## 使用场景

当 AI 处理 Jetpack 仓库中的客户端、后端、接口、文档、美术或运维任务时使用。

## 必读文档

- `docs/规范/总体工作流.md`
- `docs/规范/Git规则.md`
- 与任务模块相关的 `docs/规范/*.md`
- 相关 `docs/功能/*.md`

## 执行步骤

1. 明确任务目标、范围、交付物和验收标准。
2. 确认是否已有 Issue；没有 Issue 时按 `docs/规范/Issue规则.md` 创建或建议创建。
3. 从 `main` 创建任务分支。
4. 修改对应模块，必要时同步更新 OpenAPI 和功能文档。
5. 执行相关测试、格式化或构建。
6. 提交 PR，按模板填写测试结果、影响范围、风险和回滚。

## 输出格式

- 变更内容
- 测试结果
- 未完成事项或风险
- 关联 Issue/PR

## 注意事项

Skill 只描述执行流程。完整规则以 `docs/规范` 为准。
