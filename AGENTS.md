# AGENTS.md

## 项目是什么

Jetpack 是一个横板跳跃闯关网页游戏项目。用户可注册账号、登录、进行横板闯关，并在商店购买人物皮肤。

当前阶段是项目初始化，目标是保证客户端、后端、接口契约、文档和协作流程都具备清晰入口。

## AI 开始任务前必须阅读

1. `docs/规范/总体工作流.md`
2. `docs/规范/Git规则.md`
3. 与任务模块相关的规则：
   - 客户端：`docs/规范/客户端规则.md`
   - 后端：`docs/规范/后端规则.md`
   - 接口：`docs/规范/接口规则.md`
   - 美术：`docs/规范/美术资源规则.md`
   - 发布：`docs/规范/发布规则.md`
   - Projects：`docs/规范/Projects字段规则.md`
4. 相关功能文档，例如 `docs/功能/项目概览.md`

## 基本执行流程

1. 从 Issue 或用户任务确认目标、范围和验收标准。
2. 阅读对应 docs，不在 AGENTS.md 中寻找完整规范。
3. 从 `main` 创建符合规范的任务分支。
4. 修改代码、文档、接口契约或资源。
5. 执行必要测试和构建。
6. 创建 PR，并在 PR 中说明关联 Issue、变更内容、测试结果、影响范围、风险和回滚方式。

## 不确定情况

如果任务范围、接口字段、资源版本、数据影响、发布风险不明确，应在 Issue 或 PR 中提问，并引用相关 docs。不要静默猜测。

## 规则优先级

如 AGENTS.md、Skills、README、Issue 模板或 PR 模板与 `docs/规范` 冲突，以 `docs/规范` 为准。
