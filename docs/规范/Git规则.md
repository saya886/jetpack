# Git 规则

## 分支

- 分支从 `main` 创建。
- 一个分支处理一个明确目标。
- 分支名格式：`Issue编号-类型-任务名称`。
- 多个强相关 Issue 可写为：`Issue编号1,Issue编号2-类型-任务名称`。
- 分支名只使用中文、英文、数字、短横线 `-`、英文逗号 `,`。

示例：

```text
123-客户端-登录页面
124-后端-登录接口
125,126-登录功能-接口页面联调
```

## Commit

Commit 信息建议：

```text
类型: #Issue编号 说明
```

示例：

```text
客户端: #123 完成登录页面
后端: #124 完成登录接口
文档: #125 更新接口说明
```

## PR

PR 应说明：

- 关联 Issue
- 变更内容
- 测试结果
- 影响范围
- 风险与回滚
