# Projects 字段规则

GitHub Projects 用于跟踪 Issue 和 PR 的执行状态。

## 推荐字段

- Status：任务状态。
- Type：任务类型，只使用客户端、后端、美术、运维、文档。
- Priority：优先级。
- Start date：开始日期。
- End date：结束日期。
- Owner：负责人。

## Status 建议值

- Backlog：待排期。
- Ready：已明确范围，等待开始。
- In progress：开发或制作中。
- In review：PR 审核中。
- Done：已合并或完成。

## 使用规则

- Issue 创建后应进入 Projects。
- Status 变化应反映真实进展。
- 日期字段与 Issue 标题或正文保持一致。
- 类型字段与 `docs/规范/Issue规则.md` 保持一致。
