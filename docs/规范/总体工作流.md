# 总体工作流

项目正式修改遵循：

```text
Issue -> Branch -> 修改 -> PR -> main
```

## 原则

- Issue 是任务入口。
- `main` 只保存稳定版本，不直接提交到 `main`。
- Branch 用于完成明确目标。
- PR 是审核与合并入口。
- docs 是项目规范的唯一来源。
- AGENTS.md 和 Skills 只索引 docs，不复制完整规范。
- 规则冲突时，以 `docs/规范` 为准。

## 初始化完成标准

- 目录结构已创建。
- README.md 已说明项目用途和启动方式。
- AGENTS.md 已索引 docs。
- `docs/规范` 已包含基础规则。
- Issue 模板可用。
- PR 模板可用。
- 客户端和后端可本地启动。
